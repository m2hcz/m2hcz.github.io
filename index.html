WooCommerce Designer Pro — Unrestricted File Upload to RCE
CVE-2025-6440CriticalCVSS 9.8 · CWE-434
An unauthenticated arbitrary file upload flaw in the WooCommerce Designer Pro plugin for WordPress, in the wcdp_save_canvas_design_ajax handler: missing file-type validation lets an attacker upload a PHP payload and achieve full remote code execution on the underlying server. I published a working proof of concept; the finding, tracked as CVE-2025-6440, has since been indexed across exploit-tracking and vulnerability-intelligence platforms.
Published independently as research on a known plugin vulnerability, not a coordinated vendor disclosure — see the note at the end of this section.

File Browser — Symlink Directory Scope Bypass
CVE-2026-54094HighCVSS 7.5 · CWE-22 / CWE-59Fixed · v2.63.14
Assigned CVE-2026-54094 through GitHub's Security Advisory process. File Browser's per-user scope is enforced against lexical ../ traversal, but its HTTP handlers followed symbolic links before opening, serving, writing, or sharing a file. That let scoped users — and in some cases unauthenticated public-share recipients — read, overwrite, and share files outside the area the application was meant to enforce. The maintainers accepted the report, shipped a fix, and published the patched release.
Credit to the File Browser maintainers for the fast triage, technical validation, and responsible fix.
