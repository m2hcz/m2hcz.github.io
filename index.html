Gitea — Git LFS Cross-Repository Authorization Bypass
CVE-2026-28740 High CVSS 7.1 · CWE-639 / CWE-863 Fixed · PR #38006
Reported through GitHub's Security Advisory process and assigned CVE-2026-28740. An authorization flaw in Gitea's cross-repository Git LFS flow let a user with non-Code access improperly authorize and retrieve private LFS objects belonging to source repositories. File Browser — Symlink Directory Scope Bypass
Open Source · Go GitHub Security Advisory High · CVSS 7.5 Fixed in v2.63.11
Reported through GitHub's private Security Advisory process, this finding involved scope validation when handling linked directories and symlinks: certain operations could escape the intended boundary, allowing scoped users and public-share recipients to read and write files outside the area the application was meant to enforce. The maintainers accepted the report, shipped a fix, and published the patched release. The case underlines how critical strict path and scope validation is for any file-handling application, especially with public sharing, symbolic links, and per-user permissions on network-exposed surfaces.
Credit to the File Browser maintainers for the fast triage, technical validation, and responsible fix.
